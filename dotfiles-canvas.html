01

The boot sequence

install.sh — idempotent, safe to re-run; every step skips itself if already done. Installers are told to keep their hands off your rc files so the rcm symlinks never get clobbered.

1Homebrew

Installed if missing; handles both Apple Silicon and Intel prefixes.

2brew bundle

Every package in the Brewfile (section 02), plus the thoughtbot/formulae tap.

3fzf key bindings

Writes ~/.fzf.zsh — Ctrl-R history search and Ctrl-T file finder, sourced by zshrc.

4oh-my-zsh

Installed with KEEP_ZSHRC=yes so it never touches the managed ~/.zshrc.

5zsh-autosuggestions

Cloned into oh-my-zsh custom plugins; activated by the forge block in zshrc.

6nvm → ~/.nvm

Official installer with PROFILE=/dev/null — no rc-file graffiti. Lazy-loaded later (section 04).

7chsh → zsh

zsh becomes the login shell (added to /etc/shells if needed).

8rcup symlinks

env RCRC=$HOME/dotfiles/rcrc rcup — every dotfile linked into $HOME (section 03).

The symlink layer

rcm links the repo into $HOME as dot-prefixed files. ~/dotfiles-local takes precedence over ~/dotfiles, so personal overrides (aliases.local, zshrc.local, gitconfig.local…) win without forking the repo.

Repo → home rcrc

zshrc / zshenv / aliases→~/.zshrc · ~/.zshenv · ~/.aliases
zsh/ (configs, functions)→~/.zsh/
gitconfig / gitmessage / gitignore→~/.gitconfig · ~/.gitmessage · ~/.gitignore_global
git_template/→~/.git_template/
bin/→~/.bin/ (first on PATH)
config/herdr/config.toml→~/.config/herdr/config.toml
psqlrc · asdfrc · hushlogin · ctags→~/.psqlrc · ~/.asdfrc · ~/.hushlogin · ~/.ctags

Rules rcrc + hooks/post-up

  • Excluded from linking: README*, LICENSE, install.sh, Brewfile, CODE_OF_CONDUCT.md
  • Copied, not linked: git_template/HEAD
  • post-up hook: touches ~/.psqlrc.local so psql never errors on a missing include
  • post-up hook: warns if a legacy /etc/zshenv will scramble PATH ordering

zshrc loads functions first, then ~/.zsh/configs in pre → main → post order, then local overrides and aliases — then oh-my-zsh, version managers, prompt, and finally herdr.

06

herdr — the tmux successor

Autostarts on every interactive terminal (guards: real TTY, not already inside herdr or tmux; escape hatch HERDR_NO_AUTOSTART=1). The keymap deliberately mirrors the old tmux config — same prefix, same split characters.

Prefix & panes config/herdr/config.toml

ctrl+athe prefix (ex-tmux)
pfx+←↓↑→focus pane in any direction
pfx+xclose pane
pfx+fzoom pane

Splits — tmux dialect

pfx+%split side-by-side
pfx+"split stacked
All bindings carry the prefix on purpose — a bare % or arrow would be an always-on binding that steals keys from the shell and TUIs.

Workspaces & tabs

pfx+⇧↑/⇧↓prev / next workspace
pfx+⇧←/⇧→prev / next tab
Rose Garden retired — the custom pastel-pink theme was removed 2026-06-11; herdr now runs its stock theme. The quick keys stayed.

How git behaves

Identity, guardrails, and a hook template copied into every repo you init or clone.

Config gitconfig

identityDavid Whitlatch <davidtwhitlatch@gmail.com> — overridden per-tree: includeIf gitdir:~/doubledev/ loads a separate identity

editorcode --wait · diff.colorMoved=zebra · credentials in macOS keychain · git-lfs filter
templatescommit message = 50/72 prompt sheet (~/.gitmessage) · global ignore for .DS_Store, node_modules, .env, logs, tags…

Aliases gitconfig [alias]

git aa / apadd --all / add --patch
git ci / co / stcommit -v / checkout / status
git pfpush --force-with-lease
git branchesremote branches by recency, with author

Hook template — copied into every new repo git_template/hooks/*

  • ctags re-index on post-checkout, post-commit, post-merge, post-rewrite — runs in the background, writes .git/tags so editors get jump-to-definition for free
  • graphify auto-rebuild on post-commit and branch switches — refreshes the knowledge graph (code nodes only) when graphify-out/ exists; skips during rebase/merge/cherry-pick
  • local hook chaining — every hook (pre-commit, pre-push, commit-msg, prepare-commit-msg…) sources its counterpart in ~/.git_template.local/hooks/ first, so personal hooks extend without editing the repo
  • info/exclude — per-repo ignores seeded at init time (currently growing a claude-code-runtime block, uncommitted)

Custom commands on PATH

bin/ → ~/.bin, loaded first on PATH. The git-* scripts behave like native subcommands: git up, git co-pr 42…

Branch lifecycle

git upfetch + rebase onto origin's primary branch
git create-branch xcreate on origin, track, and check out in one move
git delete-branch xdelete locally and on origin
git rename-branch xrename locally and swap the remote branch
git merge-branchmerge the previous branch into a verified-fresh main

Daily drivers

git co-pr 42fetch and check out PR #42 as pr/42
git caamend with a refreshed timestamp
git current-branchplumbing used by prompt + other scripts
git trust-bin / ctagsmark a repo's bin/ trusted for PATH · force a tags rebuild
replace foo bar **/*.rbproject-wide find & replace via rg/ag + sed
bin/tat (tmux-attach-by-directory) ships disabled — superseded by herdr.
09

The supporting cast

psql psqlrc

NULLs render as [NULL] instead of invisible emptiness; further tuning belongs in ~/.psqlrc.local (auto-created so psql never errors).

ctags ctags · ctags.d/

Custom Elixir tag definitions (modules, macros, protocols, tests) and JS constants; excludes node_modules / vendor. Feeds the git hook indexer.

Quiet things hushlogin · asdfrc

.hushlogin silences the macOS "Last login" banner. .asdfrc lets asdf read legacy version files like .nvmrc.

The override layer ~/dotfiles-local

Nothing here requires forking: aliases.local, zshrc.local, zshenv.local, gitconfig.local, psqlrc.local, git_template.local/hooks/*, and extra zsh/configs (with pre/ and post/ load ordering) all get picked up automatically — and zshenv even warns you if you put PATH edits in the wrong one.
